How to read this:
  * Pick the SESOI = the smallest CLASSICAL QQ-violation |q| you refuse to miss
    (i.e. the most quantum-LOOKING classical order effect you still want to catch).
  * N(per order) in that row is the pre-registered powered sample size; double it
    for both asking orders, then multiply by 2 questions/session for the API budget.
  * As |q| -> 0 the M1 process becomes indistinguishable from M2 (quantum) at any
    feasible N -- this is the 'false-quantum' boundary: under-powering biases toward
    wrongly crowning the quantum model. So the SESOI must be set strictly > 0.
  * 'detect-order N' is the (much smaller) analytic N merely to DETECT the order
    effect (reject M0). It is NOT the binding constraint -- the pilot's large order
    effects already clear it. The QQ-residual resolution (the M1-vs-M2 column) binds.

Pilot linkage (honest): the n=16 pilot fixes nothing quantitative here. It rules out
M0 cheaply (order effects were large) but cannot estimate q at n=16, so we power for a
chosen SESOI on q rather than back N out of the pilot. With LLMs N is cheap, so a small
SESOI (a tight |q|) is affordable; the recovery sim's q=0.0325 row is one such choice.
This says nothing about which process real LLMs realize -- that is the live run.
